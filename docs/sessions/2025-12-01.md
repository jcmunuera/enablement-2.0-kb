## Decisions Made

### 1. Unified ERI for Multiple Options

**Decision:** When an ADR defines multiple implementation options, create ONE single ERI that documents all options.

**Context:** ERI-012 Persistence could have been 4 separate ERIs (JPA, Feign, RestTemplate, RestClient) or 1 unified.

**Result:** 1 ERI with Option A (JPA) and Option B (System API with 3 client variants) sections.

### 2. MODULE Breakdown Criteria

**Decision:** 
- **Functionally equivalent** options → 1 MODULE with internal variants
- **Functionally disparate** options → Separate MODULEs

**Application:**
- Feign/RestTemplate/RestClient → 1 MODULE (mod-017) with folders per variant
- JPA/System API → 2 separate MODULEs (mod-016, mod-017)

### 3. Version Reset

**Decision:** Internal version v7.0 becomes v1.0.0 when migrating to GitHub.

**Context:** Fresh start with complete traceability in Git.

### 4. Commit Methodology

**Decision:** 1 commit per work session, with documentation in `docs/sessions/`.

**Exceptions:** Major initiative closures generate immediate commit + tag.
